üîê How StegoChannel Works

The secret is not in what you say‚Äîit's in which posts you choose to publish.

üì§ Sender Side

  1. Enter a secret message (e.g., "HELLO")
  2. The message is converted to bits: 01001000 01000101...
  3. Write posts naturally‚Äîeach post's features encode 5 bits:
    • Length ‚â• 50 chars ‚Üí bit 1, otherwise ‚Üí bit 0
    • Has media ‚Üí bit 1, otherwise ‚Üí bit 0
    • Contains ? ‚Üí bit 1, otherwise ‚Üí bit 0
    • First word category ‚Üí 2 bits (pronoun=00, article=01, verb=10, other=11)
  4. When a post's bits match the next needed bits ‚Üí publish as SIGNAL
  5. Otherwise ‚Üí publish as COVER (camouflage traffic)

üì• Receiver Side

  1. Monitor the sender's public feed
  2. For each post, compute: hash(author + content + epoch)
  3. If hash < threshold ‚Üí it's a SIGNAL post, extract the 3 feature bits
  4. Accumulate bits until the full message is reconstructed
  5. Verify the MAC to confirm authenticity

üîë The Key Insight

Without the shared secret key, an observer cannot determine which posts are signals and which are cover. All posts look like normal social media activity. The threshold is derived from the key, so only someone with the key can identify signal posts.

‚è±Ô∏è Why So Slow?

At ~5 bits per signal post and ~25% signal rate, transmitting "HELLO" (40 bits) requires roughly 8 signal posts spread over days of normal posting. This slowness is the feature, not a bug‚Äîit makes the channel indistinguishable from normal activity.
